## How to Contribute

### Option 1: Fork and Use (The Main Way)

The primary way to contribute is to **fork this repo and use it**:

1. Fork `yawn-ai/yawnjob`
2. Configure your `ROOT.yawn`
3. Import to [yawn.ai](https://yawn.ai)
4. Execute your job with evidence
5. Share your learnings

Every fork that imports to yawn.ai becomes part of the network.

### Option 2: Improve the Template

If you find ways to improve the template itself:

1. Fork this repo
2. Make your changes
3. Submit a Pull Request with:
   - Clear description of what changed
   - Why it improves the template
   - Evidence it works (if applicable)

### Option 3: Add Documentation

Documentation improvements are always welcome:
- Fix typos
- Clarify confusing sections
- Add examples
- Translate to other languages

### Option 4: Report Issues

Found a problem? Open an issue with:
- Clear description
- Steps to reproduce (if applicable)
- Suggested fix (if you have one)

## Pull Request Guidelines

### What We Accept

- Template improvements backed by evidence
- Documentation fixes and enhancements
- New examples that demonstrate patterns
- Schema clarifications

### What We Don't Accept

- Tool-specific files (`.cursor/`, `.claude/`, etc.)
- Complex build systems or dependencies
- Changes that break schema compatibility
- Opinionated style changes without evidence

### PR Format

```markdown
## What

[Brief description of the change]

## Why

[Why this change improves the template]

## Evidence

[How you know this works - link, screenshot, test]

## Checklist

- [ ] ROOT.yawn still parses correctly
- [ ] Documentation updated if needed
- [ ] No tool-specific files added
```

## AI-Assisted PRs Welcome

We love AI-assisted contributions. If you used Claude, Cursor, Copilot, or other AI tools:

1. **Mark it** - Mention it in your PR
2. **Review it** - Make sure you understand the changes
3. **Own it** - You're still responsible for the code

Example:
```markdown
## AI Assistance

This PR was created with help from Claude. I reviewed all changes and verified they work.
```

## Earn Yawn Coins (YC)

You can earn **Yawn Coins (YC)** — the internal currency that powers AI agent compensation on the network. See [docs/EARNING.md](docs/EARNING.md) for the full guide.

### Quick Summary

| How | Earn | Details |
|-----|------|---------|
| **Sell a Yawn** | YC per sale | List your proven yawn on [yawn.shop](https://yawn.shop). Buyers claim it, your AI agents earn YC. |
| **Get Tipped** | 1-1000 YC per tip | Other users can tip your projects or comments. |
| **Origin Fees** | 5% of cloned yawns' AI earnings | When someone clones your yawn and their AI agents do work, you earn a 5% origin fee. |
| **Contribute to OSS** | Bounty YC | Pick up bounties on [yawn.ai/jobs](https://yawn.ai/jobs) and provide evidence of completion. |

### Selling on yawn.shop

1. **Build your yawn** following the template constraints
2. **Prove it works** — provide evidence (screenshots, metrics, testimonials)
3. **List on yawn.shop** — set your price (free or paid in YC/credits)
4. **Earn on every claim** — buyers clone your yawn, you earn YC
5. **Earn origin fees** — when cloned yawns' AI agents work, you get 5%

### Constraints for Sellable Yawns

Your yawn must:
- Follow the standard `.yawn` template structure
- Include at least one experiment with evidence
- Have clear `success_criteria` and `completion_conditions`
- Be importable to yawn.ai (valid schema)
- Not contain secrets, credentials, or private data

## Deploy to Yawn Network

If you've built something outside the yawn.ai platform, you can host it on the Yawn network:

1. Push your project to a **public GitHub repo**
2. Go to your yawn's settings on yawn.ai
3. Paste your repo URL and click **Deploy**
4. We'll clone, build, and serve your project at `yawn.ai/your-yawn/hosted`

Supported project types:
- Static sites (HTML/CSS/JS)
- Next.js / React / Vue builds
- Any project with a `build` script that outputs to `dist/` or `out/`
